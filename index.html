1.1.9 Project- Draw It Yourself

Our project designed using Python Turtle, called “Draw It Yourself” guides the user through a series of steps to draw their own image under the sea. The program asks the user specific questions pertaining to each animal or component of the sea and based on their answer, the program follows.

For example, the program asks the user if they want a starfish in their program and if they respond with “Yes”, then the following questions would be more detailed, gathering information about the user’s preference of color, size, position of that specific animal. This process repeats for every object in the program, until the program is complete and a picture under the sea is displayed. Each image prompts variety as the user has complete control over specific colors and sizes and positioning of each image.

1.2.5 Project- No Wifi

Our project, designed with Python Turtle is a game that allows the user to navigate through obstacles using the arrow keys. We designed this program using a set of user-defined functions to generate obstacles at random.

The more obstacles you avoid and the longer the dinosaur stays alive, and therefore the higher your score will be. In addition, the game increases in difficulty and complexity. For example, after a few seconds of staying alive, meteors will appear from the sky, an additional obstacle you need to avoid. At the end, a leaderboard will be displayed at the end with the highest scores.

1.3.1 - Fireboy

This project is an adaptation of the popular computer game called "Fireboy and Watergirl". The player must use the w, a, d keys to navigate across three different levels, and navigate to the door, which is the end of the level

If you collide with one of the red platforms, the losing screen will be displayed. However, if you reach the end of the level, then a screen with your grade and score will be displayed. Your grade is determined by the number of jewels you collect throughout the game. For example, if you collect all of the jewels except for one, you get an A-. Lastly, you must make it out of the level before a thirty second timer runs out, and if not, then you lose. As a part of the design of this game, we had to program a mechanism to make the character move,collision detection, functions to randomize the location of certain obstacles.
